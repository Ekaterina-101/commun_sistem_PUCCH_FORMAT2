# PUCCH Codes Modeling


![C++17](https://img.shields.io/badge/C++-17-blue.svg)



В системах мобильной связи 4G (LTE) и 5G (NR) устройства пользователя передают базовой станции контрольную информацию по специальному восходящему каналу — PUCCH (Physical Uplink Control Channel). Эта информация включает:
HARQ-биты — подтверждение успешного/неудачного приёма данных
CSI-биты — данные о состоянии радиоканала
За один сеанс передачи может быть отправлено 2, 4, 6, 8 или 11 бит, что требует применения помехоустойчивого кодирования для обеспечения надёжности в условиях шумного канала.


## Описание проекта


Данный проект предназначен для симуляции и анализа кодов канала передачи управляющей информации в восходящем направлении (PUCCH). Проект включает реализацию модели канала с АБГШ (аддитивный белый гауссовский шум), алгоритмов QPSK(Quadrature Phase Shift Keying) модуляции и демодуляции с мягким выходом, кодирования при помощи кодов Рида-Маллера 1го порядка и декодирования по максимуму правдоподобия, а также тестирование с использованием фреймворка Google Test.

## Требования к системе

Для сборки и запуска проекта необходимо:

- Компилятор C++ с поддержкой C++17 (g++ 7.0 или выше)
- Make
- Google Test (gtest и gtest_main)
- pkg-config (для автоматического определения путей к библиотекам)

## Сборка проекта


Для сборки всего проекта (приложения и тестов) выполните:

```bash
make
```
Создаются:
```bash
bin/pucch_codes_modeling
bin/pucch_tests
```

Для сборки только основного приложения:
```bash
make app
```
Создаётся:
```bash
bin/pucch_codes_modeling
```
Для сборки только тестов:
```bash
make test
```
Создаётся:
```bash
bin/pucch_tests
```

## Очистка проекта
Для удаления всех скомпилированных файлов:
```bash
make clean
```
Для полной пересборки:
```bash
make rebuild
```

## Запуск проекта

Запуск основного приложения:
```bash
make run_app
```
Для передачи аргументов приложению используйте переменную ARGS:

```bash
make run_app ARGS="input.json output.json"
```
Запуск тестов:
```bash
make run_test
```

Перед запуском тестов автоматически создаётся символическая ссылка pucch_codes_modeling в корневой директории проекта, указывающая на основное приложение. Это позволяет тестам запускать приложение из текущей директории. После завершения тестов ссылка удаляется.

Ручной запуск:
```bash
./bin/pucch_codes_modeling
```
или с аргументами
```bash
./bin/pucch_codes_modeling input.json output.json
```
```bash
./bin/pucch_tests
```

## Компиляция 

Проект компилируется с флагами:
-std=c++17 - стандарт C++17
-Wall -Wextra - все предупреждения
-O2 - оптимизация уровня 2
-MMD -MP - автоматическая генерация зависимостей

Для линковки используются:
-lpthread - библиотека потоков POSIX
-lgtest -lgtest_main - библиотеки Google Test

## Тестирование

Проект использует фреймворк Google Test для модульного тестирования. Тесты автоматически собирают все исходные файлы кроме main.cpp и добавляют тестовые файлы из папки test/.

Для запуска тестов выполните:
```bash
make run_test
```

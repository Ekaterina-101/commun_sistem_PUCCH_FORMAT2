## О проекте
Данный проект предназначен для симуляции и анализа кодов канала передачи управляющей информации в восходящем направлении (PUCCH).

В мобильных сетях 4G/5G устройства передают базовой станции критически важную управляющую информацию по каналу PUCCH:


| Тип информации | Описание |
|----------------|----------|
| **HARQ-ACK/NACK** | Подтверждение успешного/неудачного приёма данных |
| **CSI** | Информация о состоянии радиоканала |

За один сеанс передачи может быть отправлено **2, 4, 6, 8 или 11 бит**. Для обеспечения надёжности в условиях шумного канала применяется **помехоустойчивое кодирование** на основе **блочных кодов** со следующими характеристиками:

- Порождающая матрица **A<sub>20×13</sub>** из стандарта 3GPP TS 36.213
- Кодовые слова фиксированной длины **20 бит** для всех режимов
- QPSK-модуляция с мягкой демодуляцией (LLR)
- Мягкое декодирование методом максимального правдоподобия (ML)
- Анализ характеристик через метрику **BLER** (Block Error Rate)

## Архитектура системы
┌──────────────────┐     ┌──────────────┐     ┌──────────────┐
│ Генератор        │     │ Блочный      │     │ QPSK         │
│ случайных данных │────▶│ кодер        │────▶│ модулятор    │
└──────────────────┘     └──────────────┘     └──────────────┘
        │
        ▼
┌──────────────┐
│ Модель АБГШ  │
│ (шум σ²)     │
└──────────────┘
        │
        ▼
┌───────────────────┐     ┌──────────────┐     ┌──────────────┐
│ Мягкий QPSK       │     │ Мягкий       │     │ Расчёт       │
│ демодулятор (LLR) │────▶│ декодер (ML) │────▶│ BLER         │
└───────────────────┘     └──────────────┘     └──────────────┘
    


### Ключевые компоненты

| Компонент | Описание |
|-----------|----------|
| **Блочный кодер** | Преобразует 2/4/6/8/11 бит в кодовое слово длиной 20 бит с использованием порождающей матрицы A<sub>20×13</sub>. Для кодов меньшей длины используются подматрицы с отбрасыванием столбцов. 
| **QPSK модулятор** | Кодирует пары бит в комплексные символы созвездия QPSK: `[00→(-1-j)/√2, 01→(-1+j)/√2, 10→(1-j)/√2, 11→(1+j)/√2]` 
| **Канал АБГШ** | Добавляет гауссовский шум с нулевым средним и дисперсией σ² = N₀/2 к каждому  QPSK-символу |
| **Мягкий демодулятор** | Вычисляет LLR (Log-Likelihood Ratio) для каждого бита |
| **Мягкий декодер** | Декодирование методом полного перебора: `ĉ = arg max(c · LLR)`, где `c` — кодовое слово, `LLR` — вектор мягких решений |

## Сборка проекта
### Быстрый старт


```bash
make          # Сборка приложения и тестов
make run_app  # Запуск приложения
make run_test # Запуск тестов
```

Сборка осуществляется через **Makefile**


| Цели                                      | Назначение                  | Результат                   |
--------------------|-----------------------------|-----------------------------|
| `make`                                       | Полная сборка               | `app` + `test`              |
| `make app`                                   | Сборка основного приложения | `bin/pucch_codes_modeling`  |
| `make test`                                  | Сборка тестов               | `bin/pucch_tests`           |
| `make run_app ARGS="input.json output.json"` | Запуск с аргументами        | Выполнение с параметрами    |
| `make run_test`                              | Запуск тестов               | С управлением symlink       |
| `make clean`                                 | Очистка                     | Удаление `bin/` и `obj/`    |
| `make rebuild`                               | Пересборка                  | `clean` + полная пересборка |


## Использование
Приложение принимает входной файл input.json и сохраняет результат в result.json.


### Режим 1: Кодирование (coding)
input.json
```JSON
{
  "mode": "coding",
  "num_of_pucch_f2_bits": 4,
  "pucch_f2_bits": [1, 0, 1, 1]
}
```

result.json
```JSON
{
  "mode": "coding",
  "qpsk_symbols": [
    "0.7071+0.7071j",
    "-0.7071+0.7071j",
    "0.7071-0.7071j",
    "-0.7071-0.7071j",
    "0.7071+0.7071j",
    "-0.7071+0.7071j",
    "0.7071-0.7071j",
    "-0.7071-0.7071j",
    "0.7071+0.7071j",
    "-0.7071+0.7071j"
  ]
}
```

### Режим 2: Декодирование (decoding)
input.json
```JSON
{
  "mode": "decoding",
  "num_of_pucch_f2_bits": 6,
  "qpsk_symbols": [
    "0.682+0.712j",
    "-0.721+0.693j",
    "0.705-0.709j",
    "-0.698-0.716j",
    "0.711+0.703j",
    "-0.704+0.710j",
    "0.699-0.715j",
    "-0.713-0.701j",
    "0.708+0.706j",
    "-0.702+0.712j"
  ]
}
```

result.json
```JSON
{
  "mode": "decoding",
  "num_of_pucch_f2_bits": 6,
  "pucch_f2_bits": [1, 0, 1, 1, 0, 1]
}
```

### Режим 3: Моделирование канала (channel simulation)

input.json
```JSON
{
  "mode": "channel simulation",
  "num_of_pucch_f2_bits": 8,
  "snr_db": 4.5,
  "iterations": 100000
}
```

result.json
```JSON
{
  "mode": "channel simulation",
  "num_of_pucch_f2_bits": 8,
  "bler": 0.0237,
  "success": 97630,
  "failed": 2370
}
```

## Обработка ошибок
При некорректных входных данных приложение:
1. Выводит описание ошибки в stderr
2. Возвращает ненулевой код возврата (exit(1))
3. Не создаёт файл result.json
### Примеры ошибок:
- Недопустимое значение num_of_pucch_f2_bits (не в {2,4,6,8,11})
- Несоответствие длины массива pucch_f2_bits значению num_of_pucch_f2_bits
- Отсутствие обязательных полей в JSON
- Некорректный формат комплексных чисел в qpsk_symbols
